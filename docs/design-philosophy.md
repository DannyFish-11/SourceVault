# SourceVault Design Philosophy

## One sentence

SourceVault should feel like a truthful research instrument, not a stylish AI performance.

## What "true" means

"True" does not mean ugly, rough, or boring.
It means:
- the interface earns trust
- visual choices come from function
- structure is more important than spectacle
- the product looks like it was made to be used for years

## Emotional target

The interface should feel:
- calm
- serious
- lucid
- grounded
- exact
- patient

Not:
- hyped
- magical
- cute
- cyberpunk
- glossy
- attention-hungry

## Functional beauty

Beauty comes from:
- proportion
- alignment
- spacing rhythm
- type hierarchy
- restrained contrast
- careful grouping
- honest status indication
- repeatable component logic

Not from:
- glow
- blur
- gradients
- oversized hero visuals
- animated surfaces
- novelty styling

## Design references by category

Closer to:
- editorial tools
- archival systems
- annotation workspaces
- professional data consoles
- research software
- document-centric products

Farther from:
- AI landing pages
- crypto dashboards
- futuristic concept UIs
- generic motion-heavy SaaS products
- "copilot" clones

## Page philosophy

A SourceVault page should answer:
- what changed
- why it matters
- where it came from
- how trustworthy it is
- what I should do next

It should not try to "impress" before it helps.

## Information philosophy

Information should be:
- scannable
- layered
- traceable with evidence
- time-aware
- source-worthy
- directly reversible to source

This is why the interface should prioritize:
- rows
- tables
- timelines
- filters
- tags
- status markers
- update history
- source provenance
- comparison states

## Styling philosophy

Prefer:
- neutral backgrounds
- low-saturation accents
- sharp organization
- narrow radii
- subtle states
- almost no shadow
- consistent borders
- strong type hierarchy

## Motion philosophy

Motion should be minimal and functional.
Use it only for:
- state transitions
- loading feedback
- focus movement
- lightweight disclosure

Never use motion to create fake intelligence or emotional bonding.

## The grayscale test

Every screen must look credible in grayscale.
If the screen collapses without accent color, it is too dependent on styling tricks.

## Final principle

If a choice makes the interface look more "AI", but less credible, remove it.
If a choice makes the interface feel plainer, but more trustworthy, keep it.
