# UI Guardrails

## Primary rule

Do not build a generic AI dashboard.

## Forbidden defaults

Avoid these common failure patterns:
- glass panels
- layered neon shadows
- purple / blue / cyan marketing gradients
- giant rounded statistic cards
- floating panel grids with too much empty space
- overly thick soft shadows
- glowing focus rings unless accessibility requires them
- too many icons competing with text
- cards for everything
- charts for decoration
- huge chatbot entry boxes
- "assistant" presence in every page
- gradient borders
- pill-shaped everything
- dark mode with fake sci-fi glow

## Dashboard rules

The dashboard is a work surface, not a spectacle.
It should prioritize:
- recent changes
- important updates
- trusted-source rows
- topic movement
- pending review
- digest summaries

Do not create:
- giant KPI tile walls
- vanity charts
- hero sections
- animated "AI system status" ornaments

## Search rules

Search should feel:
- direct
- fast
- textual
- structured

Do not turn search into:
- a theatrical chat interface
- a giant center-aligned landing page
- a magical discovery canvas

## Vault rules

The vault should feel like:
- a working archive
- a saved evidence ledger
- a research repository

Do not make it feel like:
- a Pinterest board
- a social feed
- a glossy note-taking toy

## Topic page rules

Topic pages must emphasize:
- timeline
- sources
- artifacts
- trust
- event change
- source relationships

Avoid:
- decorative concept art
- oversized summary banners
- generic "AI insight" cards with no evidence traceability

## Artifact page rules

Artifact pages must clearly show:
- source
- publish time
- update time
- related topic
- trust level
- original link
- summary
- change history

The original source must always be easy to reach.

## Component rules

### Buttons
- compact
- deliberate
- no candy gradients
- no oversized CTA behavior inside working screens

### Inputs
- sharp and calm
- no glows
- no excessive depth

### Cards
- limited use
- low radius
- near-flat
- real content density

### Tables
- first-class
- sortable
- filterable
- comfortable for scanning evidence comparison

### Tags
- small
- semantic
- not decorative confetti

## Color rules

Use color for:
- state
- trust
- warning
- links
- validation
- active selection

Do not use color for:
- artificial excitement
- pseudo-intelligence
- decoration without meaning

## Tone rules

All UI copy should sound:
- clear
- calm
- exact
- slightly editorial
- slightly operational

Not:
- excited
- over-friendly
- mystical
- over-branded
- futuristic

## Review rule

When any frontend page is generated, run this question set:
1. Remove gradients — does it improve?
2. Remove shadows — does it improve?
3. Reduce radius — does it improve?
4. Replace cards with rows or tables — does it improve?
5. Does the page look more like a real tool now?

If yes, keep the simpler version.
